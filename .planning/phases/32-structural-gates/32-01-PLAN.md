Create `gates/` directory and `gates/lib/` subdirectory.

**gates/lib/gate-utils.js** -- Shared utilities for all gate scripts:
- `readYaml(filePath)` -- Read and parse a YAML file, return parsed object or null on error. Use `js-yaml` (already available in hooks/node_modules -- use `require('../../hooks/node_modules/js-yaml')` or install a symlink/copy).
- `readFile(filePath)` -- Read a file as UTF-8 string, return content or null if missing/error.
- `appendGateResult(gatesYmlPath, entry)` -- Read existing gates.yml (if any), parse, append `entry` to `history` array, write back. If file does not exist, create with `{ history: [entry] }`. Entry must include: `gate`, `timestamp` (ISO 8601 UTC), `outcome`, `evaluator`, `must_passed`, `must_failed`, `should_passed`, `should_failed`, `structural_checks`, `notes`, `overridden: false`.
- `formatChecks(checks)` -- Format an array of `{ criterion, passed, detail }` objects into a human-readable string for failure messages. Each failed criterion becomes a bullet like: `"- M1 FAILED: SCOPE.md does not exist at .expedite/scope/SCOPE.md"`.
- `computeOutcome(mustResults, shouldResults)` -- Given arrays of `{ criterion, passed, detail }`, compute outcome: all MUST pass + all SHOULD pass = "go"; all MUST pass + some SHOULD fail = "go_advisory"; any MUST fail = "hold" (for G1) or "recycle" (caller specifies). Return `{ outcome, must_passed, must_failed, should_passed, should_failed }`.
- `buildEntry(gate, evaluator, mustResults, shouldResults, holdOutcome)` -- Construct a complete gate history entry ready for `appendGateResult`. `holdOutcome` defaults to "hold".
- `printResult(entry, failedChecks)` -- Print structured JSON to stdout: `{ gate, outcome, must_passed, must_failed, should_passed, should_failed, structural_checks, failures: [...failedMessages] }`. This is how the calling skill reads the script's output.

**Important:** Use `require('../../hooks/node_modules/js-yaml')` to reuse the existing js-yaml dependency from hooks. Do NOT install a separate copy. Alternatively, since gate scripts may be called from various CWDs, use `require(path.join(__dirname, '../../hooks/node_modules/js-yaml'))` for robustness.

**gates/g1-scope.js** -- G1 scope completeness gate:

Usage: `node gates/g1-scope.js <project-dir>` where `<project-dir>` is the directory containing `.expedite/`.

Reads: `<project-dir>/.expedite/scope/SCOPE.md`, `<project-dir>/.expedite/state.yml`

MUST criteria (all must pass for go):
- M1: SCOPE.md exists at `.expedite/scope/SCOPE.md` and is non-empty (>0 bytes)
- M2: state.yml has a `questions` array with at least 3 entries
- M3: state.yml has `intent` field set to "product" or "engineering"
- M4: SCOPE.md contains a "Success Criteria" section with at least one bullet point (regex: line starting with `- ` or `* ` after a heading containing "Success Criteria")
- M5: Every question in state.yml has a non-null, non-empty `evidence_requirements` field
- M6: SCOPE.md has Decision Area sections (headings containing "DA-" or "Decision Area") where each has both "Depth" and "Readiness" content (regex scan)

SHOULD criteria (advisory, not blocking):
- S1: Every question has non-empty `source_hints` array
- S2: At least 2 unique `decision_area` values across questions
- S3: No more than 15 questions total

On completion:
1. Compute outcome via `computeOutcome`
2. Build entry via `buildEntry("G1", "g1-scope-script", ...)`
3. Append to gates.yml via `appendGateResult`
4. Print result to stdout via `printResult`
5. Exit 0 on go/go_advisory, exit 1 on hold

Each check produces a `{ criterion: "M1", passed: boolean, detail: "specific message" }` object. Failed details must be actionable: "SCOPE.md does not exist at .expedite/scope/SCOPE.md" not "M1 failed".
