Add override protocol section to all skill preambles so the LLM knows the override flow before encountering any denial

Purpose: The deny-override-retry round-trip has a known weak link -- the LLM may forget to retry after writing the override record (research estimates ~70-85% reliability for two-step corrections). Adding a preamble section gives the LLM prior knowledge of the protocol before it encounters a denial, creating two reinforcing sources of truth (preamble + denial reason). This significantly increases the retry reliability.

Content should cover:

```markdown
# Override Protocol

When you attempt to write a state file (.expedite/*.yml) and the write is **denied by a hook**, you will see a denial reason in the tool response. Follow the instructions in the denial exactly.

## Gate Override Flow

When a state transition is denied because a gate has not passed:

1. **Read the denial reason carefully.** It will name the gate (e.g., G1, G2) and provide the override record format.
2. **Write an override record to gates.yml.** Add a new entry to the history array:
   ```yaml
   history:
     # ... existing entries ...
     - gate: "G1"  # The gate named in the denial
       timestamp: "2026-03-13T12:00:00Z"  # Current ISO 8601 timestamp
       outcome: "overridden"
       override_reason: "User requested: <the user's justification>"
   ```
3. **Immediately retry the exact same Write to state.yml** that was denied. Do not proceed to other work between steps 2 and 3. The hook will recognize the override and allow the write.

## Important Rules

- **Do not skip the retry step.** After writing the override record, you MUST retry the state.yml write immediately. This is the most critical step.
- **Do not modify the override record format.** The hook expects exactly: gate, timestamp, outcome: "overridden", and override_reason fields.
- **Override records bypass gate-phase validation.** You can write an override for any gate regardless of current phase.
- **FSM transitions cannot be overridden via gates.yml.** If the denial is about an invalid phase transition (not a missing gate), the only bypass is EXPEDITE_HOOKS_DISABLED=true.
- **After 3 failed attempts**, the denial message will suggest manual intervention. Follow those instructions.

## Emergency Bypass

If enforcement is preventing all progress (e.g., a hook bug), set the environment variable:
```
EXPEDITE_HOOKS_DISABLED=true
```
This disables all hooks. Use only for debugging -- re-enable when done.
```
