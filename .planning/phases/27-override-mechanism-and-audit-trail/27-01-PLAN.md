**Create hooks/lib/denial-tracker.js** -- a CommonJS module for tracking denial counts per transition pattern. Uses a temp file at `.expedite/.denial-counts.json` (sibling of state files, derived from filePath like audit.log path derivation). The module exports:

- `recordDenial(expediteDir, pattern)` -- increments the count for a transition pattern string (e.g., "state.yml:scope_in_progress->research_in_progress"). Returns the new count. Reads/writes `.denial-counts.json` synchronously. Creates the file if it doesn't exist. Uses try/catch with fail-open (return 1 on error).
- `clearDenials(expediteDir, pattern)` -- resets count for a pattern to 0. Called when a write succeeds (optional, for cleanup). Not critical path.
- `ESCALATION_THRESHOLD` -- constant set to 3.

The file should be ~40-50 lines, CommonJS, no dependencies beyond Node builtins (fs, path).

**Modify hooks/validate-state-write.js** -- three changes:

1. **Actionable denial messages (OVRD-01):** Update all deny() calls to include explicit override instructions. The key denial paths are:

   a. **FSM invalid transition** (HOOK-01, line ~114): Change from `'State write blocked: ' + transition.error` to include: `'State write blocked: ' + transition.error + '. If you need to bypass this, set EXPEDITE_HOOKS_DISABLED=true temporarily.'` (FSM errors are structural -- no gate override applies, only env bypass).

   b. **Gate passage missing** (HOOK-02, lines ~128-137): This is the primary override target. Change the denial message to include explicit override instructions:
   ```

   ```
   Apply this pattern to BOTH gate passage denial paths (the one where gates.yml exists but gate hasn't passed, AND the one where gates.yml doesn't exist).

   c. **Checkpoint regression** (HOOK-03, line ~164): Add: `' To bypass enforcement entirely, set EXPEDITE_HOOKS_DISABLED=true.'` (Checkpoint regression is not gate-overridable).

   d. **Gate-phase mismatch** (HOOK-04, line ~203): Add: `' To bypass enforcement entirely, set EXPEDITE_HOOKS_DISABLED=true.'`

2. **Denial tracking and escalation (OVRD-04):** After each deny() call, before the deny, call `denialTracker.recordDenial(expediteDir, pattern)` where pattern is a string identifying the denial type (e.g., `'gate:' + transition.gate + ':' + parsed.phase`).    Lazy-load denial-tracker.js in the same way fsm.js and gate-checks.js are lazy-loaded (inside the state-file processing branch after non-state passthrough).

The schema validation (HOOK-04 structural checks) still runs -- only the gate-phase context check is bypassed for overrides.

   After the existing override detection block (around line 59), add a phase lookup:
   ```javascript
   if (matchedFile === 'gates.yml' && content.indexOf('outcome: "overridden"') !== -1) {
     var yaml = require('js-yaml');
     var gateMatch = content.match(/gate:\s*["']?([A-Z]\d+)["']?/);
     var reasonMatch = content.match(/override_reason:\s*["']?([^\n"']+)["']?/);

     // Read current phase for context
     var currentPhase = 'unknown';
     try {
       var statePath = path.join(expediteDir, 'state.yml');
       var stateRaw = fs.readFileSync(statePath, 'utf8');
       var stateObj = yaml.load(stateRaw);
       if (stateObj && stateObj.phase) currentPhase = stateObj.phase;
     } catch (e) { /* fail open */ }

     entry = '---\n' +
       'event: override_write\n' +
       'timestamp: "' + timestamp + '"\n' +
       'file: "' + matchedFile + '"\n' +
       'gate: "' + (gateMatch ? gateMatch[1] : 'unknown') + '"\n' +
       'override_reason: "' + (reasonMatch ? reasonMatch[1].trim() : 'not specified') + '"\n' +
       'current_phase: "' + currentPhase + '"\n';
   }
   ```

   if (matchedFile === 'state.yml') {
