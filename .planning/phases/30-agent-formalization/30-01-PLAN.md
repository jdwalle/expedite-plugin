**gate-verifier (NEW agent -- no existing template):** Create a new system prompt based on PRODUCT-DESIGN.md "Quality Gate Enforcement" section and "Semantic Gates" subsection. The gate-verifier:
- Receives a gate ID (G2-semantic, G3, or G5), the artifact path, and the scope/design context paths
- Reads the artifact and evaluates it across 4 dimensions: evidence_support, internal_consistency, assumption_transparency, reasoning_completeness
- For each dimension: provides a score (1-5), chain-of-thought reasoning, and specific examples from the artifact
- Returns a structured outcome: Go, Go-with-advisory, or Recycle
- Anti-rubber-stamp measures: must cite specific weaknesses even on Go outcomes, must provide counterargument for each passing dimension
- Include explicit anti-bias instructions: "Evaluate as if someone else produced this artifact"
- Output format: YAML with per-dimension scores, reasoning, overall outcome, and specific failure details if Recycle
