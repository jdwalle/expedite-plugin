<!-- Skill thinning pattern from PRODUCT-DESIGN.md Section "Skill-Agent Interaction" -->

Thinned skill structure (target: 100-200 lines, scope up to 400):
1. Frontmatter (name, description, allowed-tools, argument-hint)
2. State injection preamble (cat commands for relevant state files)
3. Step sequencer: for each step:
   a. Step tracking (update current_step in state.yml, write checkpoint.yml)
   b. Decision logic (which agent to dispatch, or inline state work)
   c. Agent dispatch (Agent tool by name) OR inline logic for interactive/simple steps
   d. Result validation (artifact exists, basic structure check)
   e. State update (write state.yml, checkpoint.yml AFTER agent returns)
4. Gate evaluation step redirects to gates.yml

What STAYS in skills (not delegated to agents):
- Step sequencing and routing
- State.yml reads/writes (backup-before-write pattern)
- Checkpoint.yml writes at step boundaries
- Gates.yml writes for gate results
- User interaction (AskUserQuestion, freeform prompts)
- Agent dispatch context assembly
- Agent result validation (artifact exists, basic structure)
- Override handling
- Resume/recovery preamble

What MOVES to agents:
- Business logic (research synthesis, design generation, plan decomposition)
- Complex evaluation logic (sufficiency evaluation -> agent)
- Content generation (not state management)
