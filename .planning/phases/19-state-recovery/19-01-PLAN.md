```
## When to Use

This protocol is invoked by a skill's Step 1 preamble when the `!cat .expedite/state.yml`

## Recovery Steps

1. **Scan for artifacts** in reverse lifecycle order (first valid match wins):
   [list the 4 artifact paths with their expected header patterns for content validation]

2. **Content validation** for each artifact:
   [non-empty AND contains expected header -- specific patterns per artifact type]

3. **If a valid artifact is found -- reconstruct state:**
   a. Extract project_name from the artifact header
   b. Map artifact to phase (PLAN.md -> plan_complete, DESIGN.md -> design_complete, etc.)
   c. Extract intent if available (discretionary -- from SCOPE.md or SYNTHESIS.md "Intent:" line)
   d. Extract lifecycle_id if available (discretionary -- from SCOPE.md "Lifecycle ID:" line)
   e. Write minimal state.yml using the template below
   f. Display inline notice: "State recovered from artifacts (last phase: {phase})"
   g. Log recovery event to .expedite/log.yml if log file exists

4. **If no valid artifact found -- unrecoverable:**
   Display: "No recovery source found. Run /expedite:scope to start a new lifecycle."
   Then STOP.

## Reconstructed state.yml Template
[minimal state.yml with only the fields needed for routing -- see 19-RESEARCH.md Pattern 4]

## Project Name Extraction
[header patterns per artifact -- see 19-RESEARCH.md Pattern 3]

## Recovery Event Logging
[YAML snippet for log.yml entry -- see 19-RESEARCH.md Example 4]
```

**Specific content requirements:**

1. Artifact scan order (locked decision): PLAN.md -> DESIGN.md -> SYNTHESIS.md -> SCOPE.md (reverse lifecycle order, first valid match wins)

2. Artifact paths:
   - `.expedite/plan/PLAN.md`
   - `.expedite/design/DESIGN.md`
   - `.expedite/research/SYNTHESIS.md`
   - `.expedite/scope/SCOPE.md`

3. Content validation headers (locked decision -- non-empty AND expected header):
   - PLAN.md: first line contains "Plan:" (e.g., `# Implementation Plan: ...` or `# Product Plan: ...`)
   - DESIGN.md: first line contains "Design:" (e.g., `# Technical Design: ...` or `# Product Design: ...`)
   - SYNTHESIS.md: first line is "# Research Synthesis"
   - SCOPE.md: first line contains "Expedite Scope:" (e.g., `# Expedite Scope: ...`)

4. Artifact-to-phase mapping:
   - PLAN.md -> `plan_complete`
   - DESIGN.md -> `design_complete`
   - SYNTHESIS.md -> `research_complete`
   - SCOPE.md -> `scope_complete`

5. Project name extraction (per artifact):
   - PLAN.md: text after `: ` on line 1 (e.g., `# Implementation Plan: My Project` -> `My Project`)
   - DESIGN.md: text after `: ` on line 1
   - SYNTHESIS.md: text after `Project: ` on line 2
   - SCOPE.md: text after `Scope: ` on line 1 (e.g., `# Expedite Scope: My Project` -> `My Project`)

6. Reconstructed state.yml template: Use the structure from 19-RESEARCH.md Pattern 4. Include the `# RECOVERED:` comment line with ISO 8601 timestamp. Set `phase` to the inferred phase, `project_name` to the extracted name. Set `intent` to extracted value when available, otherwise null. Set `lifecycle_id` to extracted value when available, otherwise null. All arrays (`questions`, `gate_history`, `tasks`) are empty. `research_round` defaults to 0.

7. Important note in the protocol: "Always infer `_complete` phases, never `_in_progress`. If a skill was in progress when state.yml was lost, the partial work may be incomplete. Setting `_complete` lets the user decide whether to re-run the skill."

8. Recovery event log entry format:
   ```yaml
   ---
   event: state_recovery
   timestamp: "{ISO 8601 UTC}"
   lifecycle_id: null
   recovery_source: "{PLAN.md | DESIGN.md | SYNTHESIS.md | SCOPE.md}"
   recovered_phase: "{inferred phase}"
   recovered_project_name: "{extracted name}"
   ```

**What NOT to include (scoped out by user):**
- No .bak file references or fallback
- No sentinel field (`_write_complete`) logic
- No corruption detection or YAML validation
- No backup rotation
- No schema validation
