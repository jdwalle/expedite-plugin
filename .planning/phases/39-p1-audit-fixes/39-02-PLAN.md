**AUD-012: Override detection handles all YAML quoting styles.**

In hooks/audit-state-change.js, line 63, change:
```javascript
if (matchedFile === 'gates.yml' && content.indexOf('outcome: "overridden"') !== -1) {
```
To:
```javascript
if (matchedFile === 'gates.yml' && /outcome:\s*['"]?overridden['"]?/.test(content)) {
```

Use regex instead of indexOf to match all three YAML quoting styles:
- `outcome: "overridden"` (double-quoted)
- `outcome: 'overridden'` (single-quoted)
- `outcome: overridden` (unquoted)

The regex approach is sufficient for the audit hook (non-blocking, always exit 0). No need for full YAML parsing.

**AUD-017: Gate history immutability enforcement.**

In hooks/validate-state-write.js, in the HOOK-04 section (after the existingCount calculation at ~line 236, before the "Only validate entries beyond existing count" loop at ~line 239), insert an immutability check:

```javascript
// Verify existing entries are unchanged (immutability enforcement)
for (var ei = 0; ei < existingCount; ei++) {
  if (JSON.stringify(currentGates.history[ei]) !== JSON.stringify(parsed.history[ei])) {
    deny('Gate write blocked: existing history entry at index ' + ei +
      ' was modified. Gate history is append-only and immutable.' +
      ' To fix gate history, use the recovery procedure in ref-state-recovery.md.' +
      ' To bypass enforcement entirely, set EXPEDITE_HOOKS_DISABLED=true.');
    return;
  }
}
```

This compares each existing gate entry against the proposed content. Both sides are JavaScript objects parsed from YAML via yaml.load, so JSON.stringify comparison is deterministic for these small entry objects. The check runs before the new-entry validation loop.

Also handle edge case: if proposed content has FEWER entries than existing (truncation attempt):
```javascript
if (parsed.history.length < existingCount) {
  deny('Gate write blocked: proposed gates.yml has fewer entries (' + parsed.history.length +
    ') than existing (' + existingCount + '). Gate history is append-only.' +
    ' To bypass enforcement entirely, set EXPEDITE_HOOKS_DISABLED=true.');
  return;
}
```

Insert this check right after the existingCount calculation and before the immutability loop.
