<task type="auto">
  <name>Task 1: Add step tracking to design SKILL.md (10 steps)</name>
  <files>skills/design/SKILL.md</files>
  <action>
Add `current_step` tracking to EVERY numbered step in `skills/design/SKILL.md`. Insert the tracking instruction at the VERY TOP of each step, immediately after the `### Step N:` heading line (before any existing content).

**Two patterns depending on whether the step already writes state.yml:**

### STANDALONE pattern (steps that do NOT already write state.yml: Steps 1, 2, 4, 5, 6, 7, 8)

Insert this block immediately after the `### Step N:` heading:

```markdown

**Step tracking:** Update `current_step` in state.yml (backup-before-write):
1. Read `.expedite/state.yml`
2. Backup: `cp .expedite/state.yml .expedite/state.yml.bak` (via Bash)
3. Set `current_step` to `{skill: "design", step: N, label: "LABEL"}`
4. Set `last_modified` to current timestamp
5. Write the entire file back
```

### FOLD-IN pattern (steps that ALREADY write state.yml: Steps 3, 9, 10)

For these steps, find the existing state.yml write instructions and add ONE line to the existing in-memory update list:

```markdown
   - Set `current_step` to `{skill: "design", step: N, label: "LABEL"}`
```

For Step 10 (Design Completion), ALSO add:
```markdown
   - Set `current_step` to null
```

### Exact labels for each step (use EXACTLY these):

| Step | Label (from heading) |
|------|---------------------|
| 1 | Prerequisite Check |
| 2 | Read Scope + Research Artifacts |
| 3 | Initialize Design State |
| 4 | Generate Design Document |
| 5 | Write DESIGN.md |
| 6 | Generate HANDOFF.md (Product Intent Only) |
| 7 | Revision Cycle |
| 8 | G3 Gate Evaluation |
| 9 | Gate Outcome Handling |
| 10 | Design Completion |

### Special cases:

- **Step 3 (Initialize Design State):** Writes phase transition (`design_in_progress`). Fold `current_step` into that same write.

- **Step 9 (Gate Outcome Handling):** Writes gate history. On "go" or "go_advisory", flow continues to Step 10. On "recycle", user re-enters design. Do NOT clear `current_step` here.

- **Step 10 (Design Completion):** Writes `phase: "design_complete"`. In the SAME write, also set `current_step: null`.
