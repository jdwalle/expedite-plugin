## Cross-Phase Integration (15/18 wired)

### Integration Findings

#### INT-01: `description` field not persisted to state.yml (HIGH)

**From:** Phase 4 scope skill Step 4 (collects description from user)
**To:** Phase 4 scope skill Step 1 Case B (resume flow reads description)
**Impact:** Resume routing after session interruption will misjudge position — user always routed to Step 4 even if already completed. Does not prevent lifecycle completion, but degrades resume UX.
**Fix:** Add `description` to `state.yml.template` schema and write it in Step 4's post-Round-1 state write block.
**Affected:** STATE-02, STATE-04, CTX-01

#### INT-02: Gate outcome schema mismatch (MEDIUM)

**From:** Phase 2 `state.yml.template` (defines `go | go_advisory | recycle | override`)
**To:** Phase 4 scope Step 9 (writes `go | go_advisory | hold`)
**Impact:** `hold` is undeclared in schema; `go_advisory` absent from schema comment. Status skill displays values without validation so no runtime break, but violates schema contract for Phase 6+ consumers.
**Fix:** Update schema comment to match actual outcomes, or rename `hold` → `recycle` in scope SKILL.md.
**Affected:** GATE-02, STATE-05

#### INT-03: Template path resolution inconsistency (LOW)

**From:** Phase 5 research SKILL.md Step 8 (direct relative paths)
**To:** Phase 3 researcher templates
**Impact:** Scope skill uses `Glob **/template-name` (path-agnostic); research skill uses `skills/research/references/prompt-*.md` (path-dependent). If CWD differs from plugin root, research template reads could fail.
**Fix:** Add Glob fallback pattern in research SKILL.md Step 8, consistent with scope skill.
**Affected:** RSCH-02, TMPL-04
