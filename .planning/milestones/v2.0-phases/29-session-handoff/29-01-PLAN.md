**hooks/session-stop.js (Stop hook):**

```javascript
#!/usr/bin/env node
'use strict';

// ============================================================================
// Stop Hook: Write Session Summary
// ============================================================================
// Fires at session end. Reads current state from .expedite/ files on disk and
// writes .expedite/session-summary.md with phase, skill, step, and next action.
// Non-blocking (exit code is informational only).
//
// Exit codes:
//   0 - Always (Stop hooks must never block)
// ============================================================================

// Emergency bypass
if (process.env.EXPEDITE_HOOKS_DISABLED === 'true' || process.env.EXPEDITE_HOOKS_DISABLED === '1') {
  process.exit(0);
}
```

The script must:
1. Check EXPEDITE_HOOKS_DISABLED first (exit 0 if set)
2. Read stdin JSON (handle data/end events). The Stop hook receives session metadata but we do not need it -- we read state from disk.
3. Determine the .expedite/ directory. Use `process.cwd()` to find the project root, then look for `.expedite/state.yml`. If it does not exist, exit 0 silently (no lifecycle active -- nothing to summarize).
4. Read `.expedite/state.yml` and `.expedite/checkpoint.yml` from disk using fs.readFileSync and js-yaml.
5. Build the session summary markdown with these fields:
   - **Phase**: from state.yml `phase` field
   - **Skill and Step**: from checkpoint.yml `skill`, `step`, and `label` fields. If checkpoint is all-null, write "No active skill step". Include total step counts per skill: scope=10, research=18, design=10, plan=9, spike=9, execute=7.
   - **Last action**: from checkpoint.yml `label`. If `continuation_notes` is not null, append it.
   - **Next action**: Derive from phase + skill + step. Use this logic:
     - If phase ends with `_complete` and step is "complete": "Next: /expedite:{next_skill}" (scope->research, research->design, design->plan, plan->spike or execute, execute->"Lifecycle complete")
     - If phase ends with `_in_progress` and step is a number: "Continue {skill} from step {step}: {label}"
     - If checkpoint is all-null: "Resume with /expedite:status to check position"
   - **Critical state**: Read additional context if available:
     - If `.expedite/questions.yml` exists, count questions and answered questions
     - If `.expedite/tasks.yml` exists, count completed vs total tasks
     - If neither exists or fails to read, omit this line
6. Write the markdown to `.expedite/session-summary.md` using fs.writeFileSync (overwrite any existing summary)
7. Wrap everything in try/catch. On error, write to stderr and exit 0 (fail-open).

The skill-to-total-steps lookup:
```javascript
var SKILL_STEP_TOTALS = {
  scope: 10, research: 18, design: 10, plan: 9, spike: 9, execute: 7
};
```

The phase-to-next-skill lookup for _complete phases:
```javascript
var NEXT_SKILL = {
  scope_complete: 'research',
  research_complete: 'design',
  design_complete: 'plan',
  plan_complete: 'spike',
  spike_complete: 'execute',
  execute_complete: null  // lifecycle complete
};
```
