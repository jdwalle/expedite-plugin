<objective>
Create the PreToolUse and PostToolUse hook scripts and register them in .claude/settings.json so that every .expedite/*.yml write is schema-validated and audited.

Purpose: This is the enforcement layer foundation. Without hooks, state files can be written with invalid content, wrong phase transitions, or missing fields. The PreToolUse hook makes schema violations structurally impossible. The PostToolUse hook creates an append-only audit trail. Non-state writes must pass through without delay to avoid impacting normal development workflow.

Output: Two Node.js hook scripts and settings.json hook registrations. FSM transition logic and gate passage checks are deferred to Phase 26 -- this plan implements schema-only validation.
</objective>

  <name>Task 1: Create PreToolUse validation hook</name>
  <files>hooks/validate-state-write.js</files>
  <action>
Create `hooks/validate-state-write.js` -- the PreToolUse command hook that intercepts all Write tool calls and validates .expedite/*.yml writes against their schemas.

**Script flow:**
1. Check `EXPEDITE_HOOKS_DISABLED` env var. If set to "true" or "1", exit 0 immediately (emergency bypass per Decision in PRODUCT-DESIGN.md).
2. Read JSON from stdin (Claude Code sends `{ tool_name, tool_input: { file_path, content } }`).
3. Extract `file_path` from tool_input.
4. **Non-state passthrough**: If file_path does NOT match `.expedite/*.yml` pattern (specifically: does not end with a path that matches `/.expedite/state.yml`, `/.expedite/checkpoint.yml`, `/.expedite/questions.yml`, `/.expedite/gates.yml`, or `/.expedite/tasks.yml`), exit 0 immediately. This is critical for HOOK-05 -- non-state writes must not be delayed.
5. Parse the content as YAML using js-yaml.
6. Determine which validator to use based on the filename:
   - `state.yml` -> validateStateYml
   - `checkpoint.yml` -> validateCheckpointYml
   - `questions.yml` -> validateQuestionsYml
   - `gates.yml` -> validateGatesYml
   - `tasks.yml` -> validateTasksYml
7. Run the validator.
8. If valid: exit 0 (allow the write).
9. If invalid: write JSON to stdout with `permissionDecision: "deny"` and `permissionDecisionReason` containing the specific validation errors, then exit 2.

**Error handling:**
- If stdin read fails: exit 0 (fail open -- don't block writes on hook errors)
- If YAML parse fails: exit 2 with denial reason "Invalid YAML in state file: {parse error}"
- If validator throws: exit 0 (fail open -- log to stderr for debugging)

**Implementation constraints:**
- CommonJS (require, not import)
- require('js-yaml') from relative path or node_modules
- require('./lib/state-schema') for validators
- Use process.stdin to read input (buffer all chunks, parse on 'end')
- Keep the script lean -- no unnecessary logging on success path
- Add a shebang line: #!/usr/bin/env node

**Latency considerations (HOOK-07):**
- The non-state passthrough check must happen BEFORE YAML parsing to avoid unnecessary work
- Do not read any files from disk on the passthrough path
- The entire validation path (YAML parse + schema check) should complete in under 200ms

**Create `hooks/audit-state-change.js`** -- the PostToolUse command hook that logs state file changes to an append-only audit log.

Script flow:
1. Check `EXPEDITE_HOOKS_DISABLED` env var. If set to "true" or "1", exit 0 immediately.
2. Read JSON from stdin (Claude Code sends `{ tool_name, tool_input: { file_path, content }, tool_output }`).
3. Extract `file_path` from tool_input.
4. Non-state passthrough: If file_path does NOT match `.expedite/*.yml` pattern (same check as PreToolUse), exit 0 immediately.
5. For state file writes: append a YAML entry to `.expedite/audit.log`:
   ```yaml
   ---
   event: state_write
   timestamp: "{ISO 8601 UTC}"
   file: "{filename only, e.g., state.yml}"
   ```
   For override-specific logging (when the written file is gates.yml and content contains `outcome: "overridden"`), add extra fields:
   ```yaml
   event: override_write
   timestamp: "{ISO 8601 UTC}"
   file: "gates.yml"
   gate: "{gate identifier from content}"
   override_reason: "{reason from content}"
   ```
6. Exit 0 always (PostToolUse is non-blocking -- never exit 2).

Error handling: If audit.log write fails, log to stderr and exit 0. Never block the tool call.

Add shebang: #!/usr/bin/env node
