**gates/g5-spike.js** -- G5 spike structural gate (Layer 1 of dual-layer):

Usage: `node gates/g5-spike.js <project-dir> <phase-slug>` where `<project-dir>` is the directory containing `.expedite/` and `<phase-slug>` is the phase identifier (e.g., "wave-1" or "epic-2").

Reads:
- `<project-dir>/.expedite/plan/phases/<phase-slug>/SPIKE.md`
- `<project-dir>/.expedite/plan/PLAN.md` (for TD count reference)
- `<project-dir>/.expedite/scope/SCOPE.md` (for DA references)

Uses shared utilities from `gates/lib/gate-utils.js` (same require pattern as other gate scripts).

MUST criteria (all must pass for structural go):
- M1: SPIKE.md exists at `.expedite/plan/phases/<phase-slug>/SPIKE.md` and is non-empty
- M2: Every tactical decision (TD-N) referenced in the phase section of PLAN.md has a corresponding resolution section in SPIKE.md. Extract TD identifiers from PLAN.md for the target phase, then check each appears in SPIKE.md. If no TDs found in PLAN.md for this phase, pass with advisory.
- M3: Each TD resolution in SPIKE.md has a rationale (text following the resolution that explains why). Check for "Rationale", "Reason", "Because", or paragraph content after the resolution line. If any TD resolution has no rationale text, fail with: "TD-{N} resolution has no rationale."
- M4: SPIKE.md contains implementation steps section with at least 3 steps. Look for a heading containing "Implementation" or "Steps" followed by numbered or bulleted items. Count must be >= 3.

SHOULD criteria (advisory, not blocking):
- S1: Implementation steps contain file path references (specific paths like `src/`, `lib/`, or file extensions like `.js`, `.ts`, `.md`)
- S2: Implementation steps have traceability references (TD-N or DA-N references within step descriptions)
- S3: Each TD resolution references the specific ambiguity or decision point it resolves (not just "chose X")
- S4: SPIKE.md total word count exceeds 300 words

On completion: Same pattern as other gate scripts:
1. Compute outcome via `computeOutcome` (use "recycle" as holdOutcome)
2. Build entry via `buildEntry("G5", "g5-structural-script", ...)`
3. Append to gates.yml via `appendGateResult`
4. Print result to stdout via `printResult`
5. Exit 0 on go/go_advisory, exit 1 on recycle

**IMPORTANT:** The script takes TWO arguments: project-dir and phase-slug. If phase-slug is not provided, print usage and exit 2. The phase-slug is needed to locate the correct SPIKE.md within the phase directory.

**If structural outcome is "go" or "go_advisory":** Proceed to Layer 2.

**Layer 2: Semantic verification** -- gate-verifier agent dispatch.

- `gate_id`: "G5"
- `artifact_path`: ".expedite/plan/phases/{slug}/SPIKE.md"
- `scope_path`: ".expedite/scope/SCOPE.md"

3. Each dimension has numeric `score` between 1 and 5
4. Each dimension has non-empty `reasoning` text
