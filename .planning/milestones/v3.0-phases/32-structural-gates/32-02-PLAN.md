**gates/g2-structural.js** -- G2 structural research completeness gate:

Usage: `node gates/g2-structural.js <project-dir>` where `<project-dir>` is the directory containing `.expedite/`.

Reads:
- `<project-dir>/.expedite/research/SYNTHESIS.md`
- `<project-dir>/.expedite/scope/SCOPE.md`
- `<project-dir>/.expedite/state.yml`
- `<project-dir>/.expedite/research/` directory (for evidence files)

Uses shared utilities from `gates/lib/gate-utils.js` (same `require(path.join(__dirname, 'lib/gate-utils'))` pattern as G1/G4).

MUST criteria (all must pass for go):
- M1: SYNTHESIS.md exists at `.expedite/research/SYNTHESIS.md` and is non-empty
- M2: For each DA referenced in SCOPE.md, SYNTHESIS.md contains a corresponding section (heading or content matching the DA identifier like "DA-1" or the DA name)
- M3: At least one evidence file exists in `.expedite/research/` (files matching `evidence-*.md` pattern)
- M4: SYNTHESIS.md contains readiness/sufficiency assessment content. Look for a section with "Readiness" or "READINESS" or "Sufficiency" in a heading, OR check for `.expedite/research/sufficiency-results.yml` existence. At minimum, the synthesis must address readiness.
- M5: No DA has status "INSUFFICIENT" in the readiness assessment. Parse SYNTHESIS.md or sufficiency-results.yml for per-DA status. If any DA is explicitly marked "INSUFFICIENT" (case-insensitive), this criterion fails. If status information is not parseable, pass with advisory note.

SHOULD criteria (advisory, not blocking):
- S1: Every DA has an explicit readiness status ("SUFFICIENT", "ADEQUATE", "STRONG", etc.) -- not just absence of "INSUFFICIENT"
- S2: Evidence files cover multiple sources (at least 2 distinct evidence-*.md files)
- S3: SYNTHESIS.md word count exceeds 500 words (meaningful synthesis, not a stub)

On completion: Same pattern as G1/G4:
1. Compute outcome via `computeOutcome` (use "recycle" as holdOutcome per research skill convention)
2. Build entry via `buildEntry("G2", "g2-structural-script", ...)`
3. Append to gates.yml via `appendGateResult`
4. Print result to stdout via `printResult`
5. Exit 0 on go/go_advisory, exit 1 on recycle

Each check produces actionable detail messages: "DA-2 (Performance Requirements) has no corresponding section in SYNTHESIS.md" or "DA-4 is marked INSUFFICIENT in readiness assessment -- evidence gap must be addressed before proceeding".
