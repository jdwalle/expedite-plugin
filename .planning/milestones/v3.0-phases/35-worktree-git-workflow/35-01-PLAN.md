Add worktree isolation to the task-implementer agent and update the execute skill to handle sequential merge-back after each task completes in its worktree.

Purpose: Worktree isolation prevents the task-implementer from clobbering the main working tree during execution. Each task runs in an isolated git worktree, and changes merge back only after task completion. This is the safety foundation for the per-task git commit flow in Plan 02.

Output: Updated `agents/task-implementer.md` with `isolation: worktree` frontmatter field. Updated `skills/execute/SKILL.md` with worktree merge-back handling in Step 5b.
