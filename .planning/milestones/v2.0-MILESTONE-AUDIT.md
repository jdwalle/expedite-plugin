## Integration Findings

### INT-01: Gate write path disconnected (High — known deferral)

Skills write gate results to `state.yml.gate_history`; HOOK-02 reads `.expedite/gates.yml`. The enforcement infrastructure is correct (Phase 26), but the skill write redirect is deferred to M4 skill-thinning. Workaround: override flow or `EXPEDITE_HOOKS_DISABLED=true`.

**Affected requirements:** HOOK-02 (enforcement works but blocks all gated transitions), OVRD-02 (override path works correctly)
**Deferred to:** M4 skill-thinning (v2.1+)

### INT-02: Schema outcome gap (Medium — M4 blocker)

`VALID_GATE_OUTCOMES` in `hooks/lib/schemas/gates.js` is missing `recycle`, `override`, `hold` outcomes used by skills. Must be added before M4 redirects gate writes to gates.yml.

**Affected requirements:** HOOK-04 (schema validation will reject valid skill outcomes)
**Action required:** Add outcomes to schema before M4 execution

### FLOW-01: Gate evaluation → phase transition broken

Full E2E flow: skill evaluates gate → writes to state.yml → attempts phase transition → hook reads empty gates.yml → DENIED. The user must use the override protocol to advance through any gated transition. This is the expected pre-M4 state per Phase 25-03 acknowledgment.
